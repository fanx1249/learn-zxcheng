# 如何阅读本站教程

本站教程会尽量采用一致结构，让读者能快速判断“这篇值不值得现在读”。

## 阅读顺序

建议按这个顺序浏览：

1. 先看标题和描述，确认目标是否匹配。
2. 再看“适合谁”和“准备工作”，确认自己是否能跟做。
3. 操作时跟着步骤做，每完成一段就对照检查点。
4. 遇到错误时先看“常见问题”，再去 issue 里补充反馈。

## 难度约定

| 难度 | 说明 |
| --- | --- |
| 入门 | 不要求太多背景知识，适合第一次接触 |
| 实战 | 需要跟着操作，通常会涉及多个工具 |
| 进阶 | 假设读者已经理解基础概念，重点讲设计取舍 |

## 复现优先

教程里的命令、配置和截图都应该能帮助你复现同样结果。如果你发现步骤缺失、截图过期或命令失效，可以通过 GitHub issue 反馈，也可以直接提交 Pull Request 修正。
