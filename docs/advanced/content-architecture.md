# 内容结构设计

教程站长期维护的关键不是一次性做很多栏目，而是让内容增长时仍然容易浏览、投稿和审核。

## 第一版目录策略

第一版只保留少量一级分类：

```text
beginner   入门指南
projects   实战教程
tools      工具教程
advanced   进阶专题
contribute 投稿指南
```

这样可以避免一开始就出现空栏目，也方便新作者判断教程应该放哪里。

## 什么时候新增分类

满足以下条件再新增分类：

- 已经有 5 篇以上相关教程。
- 这些教程放在现有分类里会显得混乱。
- 新分类有清晰的读者和维护者。

## 专题页的方向

后续可以用专题页组织跨目录内容，例如：

- Docusaurus 建站专题。
- Cloudflare 部署专题。
- GitHub 协作专题。
- AI 工具使用专题。

专题页不一定要移动原教程，可以用索引页串联已有文章。
