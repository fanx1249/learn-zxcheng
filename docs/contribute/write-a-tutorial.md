# 投稿指南

第一版使用 GitHub Pull Request 投稿。这样可以保留清晰的修改历史，也方便在合并前预览页面效果。

## 可以投稿什么

欢迎提交：

- 原创图文教程。
- 已有教程的补充和修正。
- 过期截图或失效命令的更新。
- 常见问题和排错经验。

暂时不接收：

- 视频课程。
- 未授权转载。
- 只包含链接集合、没有实际说明的内容。
- 无法复现或缺少上下文的笔记。

## 投稿步骤

1. Fork 仓库。
2. 新增或修改 `docs/` 下的教程文件。
3. 把截图放到 `static/img/tutorials/` 下。
4. 本地运行 `npm run build`。
5. 提交 Pull Request。
6. 根据审核意见修改。

## 内容要求

每篇教程至少需要说明：

- 适合谁。
- 准备工作。
- 操作步骤。
- 验证方式。
- 常见问题。

如果教程引用外部资料，请在文末列出来源链接。

## 作者署名

第一版先在教程正文里保留作者信息，例如：

```md
作者：Your Name
更新日期：2026-06-22
```

后续如果投稿者变多，再扩展正式作者页和作者元数据。
